# 兽医学习知识库 — 系统搭建逻辑

## 一、总体架构设计

本系统是一个基于 **Tauri 2.0** 的桌面应用程序，采用 Rust 后端 + Vue 3 前端的分层架构。选择 Tauri 而非 Electron 的核心考量是：Rust 原生执行效率更高、内存占用更小，且通过 `sqlx` 实现的编译期 SQL 检查能在构建阶段就拦截数据库查询错误，这对医学知识库的准确性至关重要。

整体架构可概括为三层：

- **数据层**：SQLite 嵌入式数据库，零部署依赖，数据存储在用户 `app_data_dir` 下
- **推理层**：Rust 实现的加权症状-疾病匹配引擎
- **展示层**：Vue 3 单页应用，通过 Tauri Command 机制与后端通信

```
┌──────────────────────────────────────────────┐
│  Vue 3 前端 (症状选择 → 推理请求 → 结果展示)   │
│          invoke('infer_diagnosis', ...)        │
├──────────────────────────────────────────────┤
│  Tauri Command Layer (18 个注册命令)           │
├──────────────┬───────────────────────────────┤
│ 知识查询模块  │  推理引擎 (engine.rs)          │
│ (CRUD/搜索)  │  加权评分 + 双维度阈值过滤       │
├──────────────┴───────────────────────────────┤
│  sqlx 0.8 (async SQLite, 编译期 SQL 检查)     │
├──────────────────────────────────────────────┤
│  SQLite 数据库 (vet_knowledge.db)             │
└──────────────────────────────────────────────┘
```

## 二、数据库建立

### 2.1 Schema 设计思路

参照兽医临床诊疗的信息组织方式，数据库采用 **实体-关系模型**，围绕"疾病"这一核心实体展开。设计原则是：每一种疾病都需要能回答临床诊疗的核心问题——**什么病、什么病因、什么症状、怎么鉴别、怎么治、预后如何**。

数据库包含 10 张实体表 + 4 张关系表 + 3 张 FTS5 全文搜索虚拟表：

**实体表（6 张）：**

| 表 | 临床对应 | 核心字段 |
|---|---|---|
| `diseases` | 疾病条目 | 中/英文名、所属分类、适用物种、概述、病因（JSON数组）、病理生理、预后、难度分级 |
| `symptoms` | 症状体征 | 中/英文名、定义、物种特异性备注（JSON） |
| `drugs` | 药物 | 中/英文名、药理分类、适应症、禁忌症、不良反应、物种剂量 |
| `diagnostic_tests` | 检查项目 | 中/英文名、分类、参考区间、结果解读、费用、报告时间 |
| `cases` | 病例 | 标题、物种、品种、年龄、体重、主诉、病史、体格检查、实验室检查、影像学、诊断、治疗、转归、要点、难度 |
| `learning_progress` | 学习进度 | 实体类型、ID、掌握度、复习次数、上次复习、下次复习 |

**关系表（4 张）：**

| 表 | 关系类型 | 关键字段 |
|---|---|---|
| `disease_symptom` | 疾病↔症状（多对多） | 频率（common/uncommon/rare）、发病阶段 |
| `disease_ddx` | 疾病↔鉴别诊断 | 鉴别要点文本 |
| `disease_treatment` | 疾病↔药物 | 治疗线别（first/second/adjunctive）、备注 |
| `disease_diagnostic` | 疾病↔检查 | 检查目的（screening/confirming/monitoring）、预期结果 |

**检索辅助（3+1 张）：**

| 表 | 类型 | 用途 |
|---|---|---|
| `diseases_fts` | FTS5 虚拟表 | 疾病全文搜索（名称+概述） |
| `symptoms_fts` | FTS5 虚拟表 | 症状全文搜索（名称+定义） |
| `drugs_fts` | FTS5 虚拟表 | 药物全文搜索（名称+分类） |
| `idx_*` | B-tree 索引 | 加速关系表 JOIN 查询 |

### 2.2 种子数据生成

种子数据由 Python 脚本（`tools/` 目录）生成，分两步写入 `001_initial.sql`：

1. **第一步**（`gen_sql.py`）：生成 30 种疾病的 INSERT 语句
2. **第二步**（`gen_sql_symptoms.py`）：追加 27 种症状 + ~170 条疾病-症状关联 + 15 条鉴别诊断关联

数据库初始化时（`db/mod.rs`），首次运行自动检测 `diseases` 表是否为空，若为空则执行种子 SQL 完成数据导入。

## 三、疾病分类体系

本系统收录的 30 种疾病按 **器官系统分类** 组织，覆盖兽医临床最常见的八大系统疾病 + 中毒急救与传染病：

### 3.1 呼吸系统（3 种）

| 疾病 | 英文名 | 核心症状 | 物种 |
|---|---|---|---|
| 肺炎 | Pneumonia | 咳嗽、呼吸困难、呼吸急促、发热、嗜睡 | 犬、猫 |
| 猫哮喘 | Feline Asthma | 咳嗽（发作性）、呼吸困难、呼吸急促 | 猫 |
| 气管塌陷 | Tracheal Collapse | 鹅叫样咳嗽（阵发性）、运动不耐受 | 犬 |

**分类逻辑**：呼吸系统疾病以 **咳嗽、呼吸困难、呼吸急促** 为核心症状群。鉴别要点在于：肺炎有发热等全身炎症反应；哮喘有可逆性气流受限且多见于猫；气管塌陷特征性表现为阵发性鹅叫样咳嗽，与气管软骨结构异常相关，多见于小型犬。

### 3.2 心血管系统（4 种）

| 疾病 | 英文名 | 核心症状 | 物种 |
|---|---|---|---|
| 心力衰竭 | CHF | 咳嗽、呼吸困难、运动不耐受、晕厥 | 犬 |
| 扩张型心肌病 | DCM | 呼吸困难、腹水、晕厥、运动不耐受 | 犬 |
| 猫肥厚型心肌病 | HCM | 呼吸困难、呼吸急促、运动不耐受、后肢瘫痪 | 猫 |
| 心包积液 | Pericardial Effusion | 呼吸困难、腹水、嗜睡 | 犬、猫 |

**分类逻辑**：心血管疾病以 **呼吸困难、运动不耐受** 为核心，但鉴别要点各不相同。心衰（MMVD）以瓣膜反流为特征；DCM 表现为心室扩张伴收缩功能降低，大型犬高发；HCM 是猫最常见的心脏病，以心室壁向心性肥厚为特征，可继发主动脉鞍血栓导致突发后肢瘫痪；心包积液以心脏压塞为危急表现。

### 3.3 泌尿系统（4 种）

| 疾病 | 英文名 | 核心症状 | 物种 |
|---|---|---|---|
| 急性肾损伤 | AKI | 厌食、呕吐、少尿/无尿、脱水 | 犬、猫 |
| 慢性肾病 | CKD | 多饮多尿、体重下降、厌食、脱水 | 犬、猫 |
| 尿路感染 | UTI | 尿频、血尿 | 犬、猫 |
| 猫下尿路疾病 | FLUTD | 尿频、血尿、呕吐 | 猫 |

**分类逻辑**：泌尿系统疾病的核心鉴别维度是 **病程急缓 + 肾功能指标**。AKI 起病急骤，肾小管上皮细胞急性坏死，GFR 急剧下降；CKD 为不可逆的进行性肾单位丢失，以多饮多尿为突出表现；UTI 以下尿路刺激征（尿频、血尿）为主；FLUTD 是猫特有的下尿路综合征，以特发性膀胱炎最常见，与应激密切相关。

### 3.4 消化系统（5 种）

| 疾病 | 英文名 | 核心症状 | 物种 |
|---|---|---|---|
| 胰腺炎 | Pancreatitis | 呕吐、腹痛（祈祷姿势）、厌食 | 犬、猫 |
| 胃肠炎 | Gastroenteritis | 呕吐、腹泻、厌食、脱水 | 犬、猫 |
| 异物梗阻 | Foreign Body | 呕吐、腹痛、厌食 | 犬、猫 |
| 肝衰竭 | Liver Failure | 呕吐、厌食、黄疸、癫痫 | 犬、猫 |
| 炎症性肠病 | IBD | 呕吐、腹泻、体重下降 | 犬、猫 |

**分类逻辑**：消化系统疾病以 **呕吐、腹痛、腹泻** 为核心症状群。鉴别要点：胰腺炎以剧烈腹痛（犬呈祈祷姿势）伴 fPLI 升高为特征；胃肠炎为急性自限性黏膜炎症；异物梗阻有吞食史且影像学可见异物；肝衰竭以黄疸、凝血障碍、高氨血症为三联征；IBD 为慢性持续性肠道免疫介导性炎症，需组织学确诊。

### 3.5 内分泌系统（4 种）

| 疾病 | 英文名 | 核心症状 | 物种 |
|---|---|---|---|
| 糖尿病 | Diabetes Mellitus | 多饮多尿、多食、体重下降 | 犬、猫 |
| 库欣综合征 | Hyperadrenocorticism | 多饮多尿、多食、瘙痒、体重增加 | 犬、猫 |
| 猫甲状腺功能亢进 | Feline Hyperthyroidism | 多食、体重下降、多饮多尿 | 猫 |
| 甲状腺功能减退 | Hypothyroidism | 体重增加、嗜睡、瘙痒 | 犬 |

**分类逻辑**：内分泌疾病的鉴别围绕 **代谢率变化** 展开。糖尿病以胰岛素不足/抵抗导致的高血糖为核心，犬多为 I 型（自身免疫性 β 细胞破坏），猫多为 II 型（胰岛素抵抗）；库欣综合征以皮质醇过多为特征，表现为蛋白质分解增加（皮肤变薄、肌肉萎缩）、肝糖异生增加；猫甲亢是老年猫最常见的内分泌病，T3/T4 升高导致基础代谢率增加，特征性地表现为 **多食但体重下降**，可继发 HCM；甲减则相反，以代谢率降低导致体重增加、怕冷、嗜睡为特征。

### 3.6 中毒与急救（4 种）

| 疾病 | 英文名 | 核心症状 | 物种 |
|---|---|---|---|
| 乙二醇中毒 | Ethylene Glycol | 呕吐、厌食、少尿、癫痫 | 犬、猫 |
| 巧克力中毒 | Chocolate Toxicity | 呕吐、呼吸急促、流涎、癫痫 | 犬 |
| 百合中毒 | Lily Toxicity | 呕吐、厌食、少尿 | 猫 |
| 布洛芬中毒 | Ibuprofen Toxicity | 呕吐、厌食、腹泻、脱水 | 犬、猫 |

**分类逻辑**：中毒性疾病按 **毒物种类 + 靶器官损伤** 分类。乙二醇（防冻剂）代谢为草酸钙结晶沉积于肾小管导致 AKI，猫致死剂量仅 1.5ml/kg；巧克力中的可可碱抑制腺苷受体导致中枢兴奋和心律失常，黑巧毒性最高；百合对猫有剧毒，任何部位（包括花粉和水）均可导致 AKI，8h 内积极治疗可挽救；布洛芬通过 COX-1 抑制导致胃黏膜损伤和肾血流减少，猫极为敏感。

### 3.7 传染病（4 种）

| 疾病 | 英文名 | 核心症状 | 物种 |
|---|---|---|---|
| 犬细小病毒病 | Parvovirus | 呕吐、腹泻、厌食、黏膜苍白 | 犬 |
| 犬瘟热 | Canine Distemper | 呕吐、鼻液、癫痫（晚期） | 犬 |
| 猫泛白细胞减少症 | Feline Panleukopenia | 呕吐、腹泻、厌食、黏膜苍白 | 猫 |
| 猫传染性腹膜炎 | FIP | 厌食、体重下降、腹水、黄疸 | 猫 |

**分类逻辑**：传染病按 **病原体类型 + 靶系统** 分类。细小病毒（CPV-2/FPV）攻击分裂活跃的肠隐窝上皮和骨髓，导致出血性肠炎伴泛白细胞减少，未免疫幼犬死亡率 91%；犬瘟热（CDV）为多系统感染，神经损伤可为不可逆；FIP 是猫冠状病毒突变引起的免疫介导性血管炎，分为湿性（渗出液）和干性（肉芽肿），传统观点认为预后极差。

### 3.8 外科/急诊（2 种）

| 疾病 | 英文名 | 核心症状 | 物种 |
|---|---|---|---|
| 胃扩张-扭转 | GDV | 呕吐、腹痛、呼吸困难、黏膜苍白 | 犬 |
| 子宫蓄脓 | Pyometra | 厌食、多饮多尿、呕吐 | 犬、猫 |

**分类逻辑**：外科急诊以 **急性危及生命** 为特征。GDV 是大型深胸犬的致命急症，胃扭转导致贲门和幽门闭塞，静脉回流受阻致胃壁缺血坏死，术前死亡率 15-30%；子宫蓄脓多发生于发情后 4-8 周的未绝育母犬，孕激素导致子宫内膜增生、宫颈关闭，细菌在封闭子宫内繁殖，手术切除预后极好。

## 四、症状体系

### 4.1 27 种症状体征的分类

症状按 **系统归属 + 临床意义** 组织为 6 大类：

**呼吸系统症状（4 种）**：咳嗽、呼吸困难、呼吸急促、鼻液
- 咳嗽需区分干咳（气管/支气管来源）与湿咳（肺实质来源）
- 猫张口呼吸几乎总是急症信号

**消化系统症状（4 种）**：呕吐、腹泻、腹痛、厌食
- 呕吐需与反流鉴别（被动 vs 主动）
- 犬祈祷姿势高度提示胰腺炎

**泌尿系统症状（4 种）**：多饮多尿、少尿/无尿、尿频、血尿
- PU/PD 的鉴别诊断在犬为糖尿病/CKD/库欣，在猫为糖尿病/CKD/甲亢

**全身性症状（5 种）**：体重下降、发热、嗜睡、脱水、多食
- 老年猫多食伴体重下降高度怀疑甲亢

**心血管/神经症状（5 种）**：腹水、晕厥、运动不耐受、黏膜苍白、后肢瘫痪
- 猫突发后肢瘫痪 + 皮温低 + 无脉搏 → 高度怀疑主动脉鞍血栓

**其他（5 种）**：黄疸、流涎、瘙痒、癫痫发作、鼻液

### 4.2 症状频率分级

每种疾病-症状关联标注 **频率** 和 **发病阶段**：

| 频率 | 权重 | 临床含义 |
|---|---|---|
| `common` | 1.0 | 该疾病大多数病例会出现的症状（>60%） |
| `uncommon` | 0.6 | 约 30-60% 病例出现 |
| `rare` | 0.3 | <30% 病例出现 |

发病阶段标注为：`全程`、`进展`、`发作`、`早期`、`晚期`、`少尿期`、`休克期`、`湿性` 等。

## 五、推理引擎

推理引擎是系统的核心诊断逻辑，模拟兽医临床的 **鉴别诊断思维过程**。

### 5.1 算法原理

输入：患宠的 **症状列表 + 物种**

对每种候选疾病计算两个维度的覆盖率：

1. **疾病覆盖度**（`disease_coverage`）：该疾病的症状库中，被患者症状匹配的比例（加权计算）
   - `weighted_score / total_weight`，权重由频率决定（common=1.0, uncommon=0.6, rare=0.3）

2. **输入覆盖度**（`input_coverage`）：患者输入的症状中，被该疾病覆盖的比例
   - `matched_count / input_count`

**综合评分** = 疾病覆盖度 × 输入覆盖度

### 5.2 阈值过滤

仅当以下两个条件同时满足时才纳入结果：
- 综合分 ≥ 0.25
- 输入覆盖度 ≥ 30%

这确保了一个只有 1 个常见症状的疾病不会因为"完美匹配了那 1 个症状"就被排在首位——它必须覆盖用户输入的足够比例的症状。

### 5.3 为什么需要双维度

原算法只有疾病覆盖度，会导致严重问题：假设用户选了"体重下降 + 癫痫发作"，某个只有"体重下降"作为 common 症状的疾病会得到 100% 的疾病覆盖度（因为它唯一的症状被匹配了），从而错误地排在第一位。

加入输入覆盖度后，该疾病最高只能得到 50% 的输入覆盖度（只匹配了 2 个中的 1 个），综合分被压在 0.5 以下。如果该疾病症状库较大，综合分更低，会被过滤掉。

## 六、系统启动流程

```
启动 → Tauri Builder
  ├── 初始化插件（shell, fs）
  ├── setup 阶段
  │   ├── db::init() → 创建 app_data_dir
  │   │   ├── 建表（schema.sql）
  │   │   └── 若 diseases 表为空 → 导入种子数据（001_initial.sql）
  │   ├── 注册 DbPool 到 Tauri State
  │   └── 注册 InferenceEngine 到 Tauri State
  └── 注册 invoke_handler（18 个 command）
      → 启动 WebView → 加载 Vue SPA
```

前端通过 `invoke('command_name', { params })` 调用后端，后端通过 Tauri State 获取数据库连接池和推理引擎实例，执行 SQL 或推理计算后返回序列化的 JSON 结果。
